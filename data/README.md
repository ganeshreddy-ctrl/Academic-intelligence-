# Data layout

Canonical tables are grouped by **domain**. `scripts/load_duckdb.py` loads every file
under `canonical/**` (any subfolder) into DuckDB, naming each table by its filename —
so the folders organise the data on disk without changing any table name or query.
Per-table column reference: [`../docs/data-dictionary.md`](../docs/data-dictionary.md).

- `courses.csv` — the 63-course catalogue (11 stacks) with course IDs, prereqs, ingest status. Generated by `scripts/build_checklist.py`.
- `canonical/` — flattened, queryable tables (committed), by domain:
  - **`delivery/`** — `delivered_niat.parquet`, `delivered_sessions.parquet`, `sessions.csv` (what actually ran). Built by `build_delivered.py` / `build_operational.py`.
  - **`feedback/`** — `session_feedback.csv` (per-session ratings). Built by `build_operational.py`.
  - **`instructors/`** — `instructor_sessions.csv` (per-instructor stats). Built by `build_operational.py`.
  - **`planning/`** — `designed_course_plan.csv`, `designed_sequence.csv`, `universities.csv`, `planning_standards.csv`, `scheduling_rules.csv` (the designed HLID/Prod plan + standards). Built by `build_designed.py`.
  - **`content/`** — `course_content.csv`, `reading_materials.csv`, `objective_questions.csv`, `coding_questions.csv`, `editorials.csv`, `tag_content_map.csv` (learning material), keyed by `course_id`/`unit_id`. Built by `build_content.py` / `build_canonical.py`.
  - **`subjects/`** — `subject_tags.csv`, `subject_tags_supplement.csv`, `course_crosswalk.csv` (local course name ↔ NxtWave subject). Built by `build_subject_tags*.py` / `build_course_crosswalk.py`.
  - **`issues/`** — `issues.csv` (RCA log). Built by `build_issues.py`.
- `reports/` — derived one-off reports (e.g. `unmapped_courses.csv`); not loaded into the store.
- `raw/` — source dumps (xlsx→CSV, JSON). **Gitignored** (~235MB); regenerable from the source files. Only the ingestion *input* — not needed to use the committed canonical data.
- `aip.duckdb` — unified DuckDB store. **Gitignored** but **rebuildable from git alone** (no raw): `python scripts/load_duckdb.py` loads every `canonical/**` file + `courses.csv`.

## Feedback → content link (RCA)

`session_feedback.unit_ids` joins to content `unit_id` (reading/objective/coding). This is the wire from a low-rated / negatively-reviewed session to the exact content units behind it — the basis of the root-cause loop.

## Scheduling & feedback dataset

Source: `Clickup - 86d3mmcgv - Semester 1 & 2 Scheduled Session Details.xlsx`
(dumped to `data/raw/scheduling/`, gitignored).

**Scope / considerations applied when the export was built:**
- Semester 1 & Semester 2 sessions only.
- NIAT **Offline** institutes only.
- All session types **excluding BREAK**.
- **Excluded** the NIAT KKH institute.
- Includes `LP_RESOURCES` and `LP_QUIZ` units that are linked to a session.

**Sheets:**
| Sheet | Rows | Contents |
|---|---|---|
| Scheduled Session Details | 239,372 | institute, batch_section, semester, session times, type (LECTURE/PRACTICE/EXAM), unit_id, resource_type |
| NIAT Scheduled Session Details | 157,502 | + instructor name/category, session status, week status |
| Quantitative Feedback | 6,945 | session_understanding_rating, teaching_quality_rating, total_feedbacks |
| Qualitative Feedback | 8,402 | positive / neutral / negative feedback **text** (2,706 have negative comments) |

**Notes:**
- Datetime columns are Excel serial numbers — convert on normalization.
- Qualitative pos/neu/neg columns hold comment **text**, not counts.
- ~53K sessions have unmarked status (`-`).
- Join key to content: feedback carries `session_id` + `unit_id`; canonical content carries `unit_id`.
